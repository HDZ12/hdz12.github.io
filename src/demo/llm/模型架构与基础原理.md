## GPT 和 BERT 的架构有什么区别？各自适用于什么场景？
### 核心架构差异

| 维度 | GPT（Decoder-only） | BERT（Encoder-only） |
|--------|---------------------|----------------------|
| 注意力方向 | 单向（因果掩码，只看左侧上文） | 双向（可同时看左右上下文） |
| 训练目标 | 因果语言模型（Causal LM）：预测下一个 Token | 掩码语言模型（MLM）：预测被 `[MASK]` 遮挡的 Token |
| 核心模块 | Masked Self-Attention + FFN | Bidirectional Self-Attention + FFN |
| 输入可见范围 | 当前 Token 只能看到历史 Token | 每个 Token 可看到整个句子 |
| 擅长任务 | 文本生成、对话、代码生成 | 文本理解、分类、检索、NER |
| 推理方式 | 自回归（逐 Token 生成） | 一次性编码整个序列 |
| 代表模型 | GPT-3、GPT-4、Llama、Qwen | BERT、RoBERTa、ALBERT、DeBERTa |

## 注意力示意

### GPT（因果注意力）

```text
位置1 → ✓ ✗ ✗ ✗
位置2 → ✓ ✓ ✗ ✗
位置3 → ✓ ✓ ✓ ✗
位置4 → ✓ ✓ ✓ ✓
```

使用 **Causal Mask（因果掩码）**，禁止关注未来 Token。

### BERT（双向注意力）

```text
位置1 → ✓ ✓ ✓ ✓
位置2 → ✓ ✓ ✓ ✓
位置3 → ✓ ✓ ✓ ✓
位置4 → ✓ ✓ ✓ ✓
```

所有位置之间都可以互相注意。

## 本质区别

GPT 学习的是：

\[
P(x_t \mid x_1,x_2,\ldots,x_{t-1})
\]

即：

> 根据过去预测未来。

BERT 学习的是：

\[
P(x_{\text{mask}} \mid x_1,\ldots,x_{mask-1},x_{mask+1},\ldots,x_n)
\]

即：

> 根据上下文恢复缺失词。

因此：

- **GPT = 生成模型（Generative Model）**
- **BERT = 表征模型（Representation Model）**
### 为什么当前大模型主流选择 Decoder-only？
统一范式：生成式框架可以通过 prompt engineering 统一处理理解和生成任务
扩展性：Decoder-only 架构在 scaling law 下表现更优
训练效率：每个 token 都参与 loss 计算，数据利用率高（BERT 只有约 15% 的 [MASK] token 贡献 loss）
